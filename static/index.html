Architecture & Logic

ScamShield utilizes a lightweight, local-first ML architecture designed to run on device and detect common payment scam strategies across regional Indian languages.

1. Rule-Based Script & Language Detector

We analyze the Unicode codepoints of incoming text to determine which native Indian script is present:

  • Devanagari (Hindi/Marathi): Range \u0900 - \u097F
  • Tamil: Range \u0B80 - \u0BFF
  • Telugu: Range \u0C00 - \u0C7F
  • Bengali: Range \u0980 - \u09FF

For transliterated Latin script (e.g. Hinglish, Tanglish), we perform a dictionary lookup matching highly distinct words such as karo, pannunga, cheyyandi, koren to guess the language.

2. Machine Learning Classifiers

We employ a dual-model TF-IDF Vectorization pipeline:

  • Model A (Binary Scam Classifier): Combines word n-grams (1-2) and character n-grams (3-5) to identify spam signals. Computes probability using a calibrated Logistic Regression model.
  • Model B (Multi-Label Tactic Classifier): Evaluates the message against 6 distinct scam vectors. Runs 6 independent binary models to score and flag specific manipulation elements.

3. Local Interpretability & Evidence Extraction

By extracting the TF-IDF feature coefficients directly from the trained Logistic Regression heads, we identify which terms in the text had the highest mathematical influence on the classification. This allows the application to isolate the exact scam word or trigger (e.g. "otp", "block", "pay") and display it as evidence.

4. Configurable explanations

Explanations are decoupled from code. Tactic descriptions and plain-english text are pulled from language-specific JSON configuration files under data/language_packs/. This makes localizing the detector's responses into other scripts a matter of config editing.

Extending ScamShield to New Languages

Adding support for a new language requires zero code modifications. Follow these brief steps:

  1. Create a JSON language pack named <language>.json under the data/language_packs/ directory.
  2. Define the language_name, romanized_words (common transliterated terms), tactic_templates (scam sentences for each tactic in native & romanized), and legit_templates.
  3. Update the Unicode script range mapping inside detector.py if the language uses a new native script.
  4. Run dataset_generator.py and train.py to retrain the models with the new samples automatically.

For more detailed information, see the Adding a Language Guide.
